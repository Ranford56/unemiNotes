# Ingeniería de Software y Modelos del Proceso
El software es esencialmente un producto, por lo que requiere de fases para su desarrollo, una estructura y metodologías que permiten un desarrollo organizado del producto. Así llegamos a la definición de ingeniería de software, esta busca el desarrollo de programas concentrándose en la calidad de dicho programa y la administración del proyecto mediante fases.
## Definición de software
Podemos entender que es el software definiéndolo como:
- **Instrucciones** que al ejecutarse generan una función con una salida esperada
- **Estructuras de datos** que permiten el manejo de datos de manera adecuada 
- **Información descriptiva** que describe la operación y el uso del programa (un README.md)

Pero aunque estas descripciones hagan un símil entre el software y un producto, existen diferencias que se deben remarcar:
- El software se desarrolla, no se manufactura
- No se desgasta, se deteriora
- El software se construye para uso individualizado
### Tipos de Software
- #### Software de Sistemas
	Sistemas Operativos, compiladores, todo lo que da soporte a otro soporte y de algo grado de interacción con el hardware
- #### Software de Aplicación
	Software Desarrollado para resolver problemas de negocios
- #### Software de Ingeniería y Ciencias
	Programas que simulan procesos complejos usando matemáticas
- #### Software Incrustado
	API, software que busca controlar un producto o sistema 
- #### Software de lineal de productos 
	Word, Excel y sus alternativas le da una funcionalidad de terminada al usuario
- #### Aplicaciones web 
	Tienen capacidad de computo, permiten operar cuentas bancarias, compras, juegos, etc. Busca la comodidad, rapidez y vistosidad
- #### Software de Inteligencia Artificial 
	Usa redes neuronales, ósea, algoritmos no numéricos para la resolución de problemas 
## Ingeniería de Software
La ingeniería de Software entonces puede ser definida como un enfoque sistemático, disciplinado y cuantificable al desarrollo, operación y mantenimiento de software, es decir, es una ingeniería que se centra en todo lo que tiene que ver con la creación, administración y uso de software pero esforzándose en mantener organización y calidad.
### Capas de la ingeniera de software
#### Herramientas
Son las que ofrecen apoyo para el proceso y para los métodos, al integrarse se obtiene la ingeniería de software asistida por computadora, permitiendo un mejor desarrollo de software.
#### Métodos 
Es la experiencia del ingeniero de software y las habilidades que conllevan dicha experiencia, no solo en el ámbito técnico sino habilidades como comunicación, modelación y la construcción del software  
#### Procesos
Une todas las capas por ende, es la capa encargada de la administración del proyecto
#### Compromiso de calidad 
La ingeniería de software esta ligada con la búsqueda de la mayor calidad posible en el desarrollo, no hay de otra.

La unión de estos elementos se denomina "modelo", este busca la mejora del desarrollo de software apoyando el diseño, evolución y especificación ya que la sociedad se basa mas y mas en software y resulta mas barato a largo plazo ya que el software esta bien estructurado y es mas fácil de arreglar o modificar

### Marco Conceptual 
La ingeniería de software tiene como antecedente a la ingeniería de sistemas, la cual busca la comprensión de sistemas complejos para abordar su modificación de forma que se mejore el funcionamiento interno para mejorar su eficiencia.
Este es el método de la ingeniera de sistemas:
- Definición del problema
- Compresión del sistema
- Elaboración de alternativas
- Elección de alternativa
- Puesta en practica
- Evaluación del impacto del cambio 

A diferencia de la ingeniera de sistemas, la ingeniería de software se basa en los sistemas informáticos o donde el software es importante.
Este es el modelo: 
- Especificación de Software
- Desarrollo de Software
- Validación de Software
- Evolución de Software
### Ética en la ingeniería de software
La ingeniería de software se desarrolla en un ambiente social con caracteres legales que limitan la libertad de los involucrados en un proyecto. Además de responsabilidades legales, existen ciertos estándares que solo se basan en el concepto de "profesionalismo" (lo que sea que sea eso), las mas básicas son: 
- #### Confidencialidad
Se debe respetar la confidencialidad de tus empleadores independientes de acuerdos legales
- #### Competencia
(En el ámbito de entendimiento de un tema, no competencia laboral) uno debe saber su capacidad, por lo que no se debe aceptar trabajo que exceda tus habilidades
- #### Derechos de propiedad intelectual
Debes conocer las leyes que rigen el uso de propiedad intelectual y debes ser cuidadoso en respetar la propiedad intelectual de otros
- #### Mal uso de computadoras
No se debe emplear las habilidades técnicas para usar incorrectamente las computadoras de otros individuos (no meter virus a otras computadoras de gente que sabe menos, este es mas un requerimiento moral)
  
## El Proceso de Software
Aquí es donde ocurre la magia, esta fase contiene el desarrollo en si del software además de la administración de dicho desarrollo mediante un conjunto de actividades sistematizadas en base a un modelo, cabe recalcar que este modelo no es algo rígido, el modelo es un enfoque adaptable que permite la modificación del mismo para moldearse mejor a los objetivos únicos de cada proyecto. 

### Estructura del proceso 
La estructura general de un proceso de ingeniería de software es el siguiente: 
- #### Comunicación
Es la primera parte del proceso, la conversación con el cliente para poder entender los objetivos, requerimientos, características y funciones del software
- #### Planeación
La planeación es la fase mas importante del proyecto, en este se crea un mapa en el que se definen las tareas que se deben realizar, en que orden, los riesgos probables, recursos necesarios, etc. 
- #### Modelado
El modelado, de manera similar a la planeación, es la creación de un mapa (mas similar a un bosquejo arquitectónico técnico) que planea la estructura técnica del proyecto, la interacción de diferentes funcionalidades, la estructura de las bases de datos y como influyen en le proyecto, etc. 
- #### Construcción
La generación del código que crea las funcionalidades y el software en si 
- #### Despliegue
La entrega del software realizado (ya sea completa o mayormente terminado) la consumidor y este ofrece la retroalimentación.

### Flujos de procesos basicos 
![[s1ingSoft.png]]
- #### Flujo de proceso lineal
Se ejecuta en secuencia las cinco actividades desde la comunicación hasta el despliegue
- #### Flujo iterativo
Repite las actividades mas veces antes de seguir con la siguiente actividad
- #### Flujo de proceso evolutivo 
Se ejecutan de manera circular, cada iteración lleva a un software mas completo 
- #### Flujo de proceso paralelo 
Las actividades se llevan a cabo en paralelo unas con otras

### MPS (Mejora del Proceso de Software)
Estos modelos presentados no aseguran que todo software creado bajo ellos sean perfectos y que no vayan a tener fallas, estos modelos solo buscan tener un mínimo control de calidad y de organización en los proyectos, pero vuelve a ser evidente que cada proyecto es único y es necesario un modelo de proceso igual de único basado en sus necesidades.
#### Enfoques del MPS
La mejora del Proceso de Software define: 
- Un conjunto de características que deben presentarse si se quiere lograr un proceso de software efectivo
- Un método para valorar si dichas características están presentas
- Un mecanismo par resumir los resultados de cualquier valoración 
- Una estrategia auxiliar para implementar características que hagan falta

Todos estos son indicadores de calidad del proceso que permiten una manera de medid la calidad del producto (el software) generado. Diferentes marcos conceptuales de mejora del proceso de software tiene implementan diferentes indicadores de calidad.
Los enfoques de evaluación mas famosos para la mejora del proceso del software son :
- Método de evaluación del estándar CMMI para le proceso de mejora (SCAMPI)
- Evaluación basada en CMM para la mejora del proceso interno (CBA IPI)
- SPICE (ISO/IEC 15504)
- ISO9001:2000 para software
## Desarrollo de Software
El desarrollo de software no es solo crear un programa que funcione y que lo hayas hecho de manera organizada, sino que un sistema de software desarrollado profesionalmente debe venir con una documentación, archivos de configuración, sitios web para conseguir el software, etc. Es decir, tiene que estar dado todo en bandeja de plata al usuario. 
Entonces existen dos tipos de software que los ingenieros de software generan: 
- Productos genéricos: un software que cualquiera puede comprar 
- Productos personalizados un software que esta hecho con especificaciones dadas por un cliente en concreto 

#### Ciclo de vida del desarrollo de sistemas (SDLC)
Digamos que después de mil cosas, despliegas tu propio software, ahora que?
Bueno, debes mantenerlo vivo, actualizado, escuchar la retroalimentación que ofrecen tus usuarios o tus empleadores en caso e que cambios se deban hacer, estar atento a bugs que necesiten reparación.
Por lo que se tiene siete fases (mas) del ciclo de desarrollo de sistemas:
- ##### Identificación de los problemas, oportunidades y objetivos
- ##### Determinación de los requerimientos de información del factor humano 
- ##### Análisis de las necesidades del sistema
- ##### Diseño del sistema recomendado
- ##### Desarrollo de documentación del software 
- ##### Prueba y mantenimiento del sistema 
- ##### Implementación y evaluación del sistema
